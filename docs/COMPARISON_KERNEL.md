# Comparison Kernel

## Core Law

No lens, no useful truth.

## Kernel Formula

`Compare(A, B, Lens, Context)`

Produces:

- first-principle roots
- causal chains
- hidden similarity
- hidden difference
- failure modes
- conditional judgment
- final memorable line

## Processing Steps

### 1. Define

Name A and B clearly.

Bad:

`Compare success and life`

Better:

`Compare money vs time through life control`

### 2. Declare Lens

The lens tells the mind what dimension to compare.

Example:

`Money vs Time through life control`

### 3. Set Context

The same comparison changes under different contexts.

Example:

Money vs Time under survival pressure is different from Money vs Time after financial stability.

### 4. Reduce

Find the first-principle root of each side.

Example:

Money = stored exchange power.  
Time = irreversible life-capacity.

### 5. Map

Map each causal chain.

Money:

`value -> exchange -> access -> options -> protection -> leverage`

Time:

`existence -> attention -> action -> experience -> relationship -> life`

### 6. Reveal Hidden Similarity

Money and time are both allocation systems.

### 7. Reveal Hidden Difference

Money can be regained. Time can only be spent.

### 8. Identify Failure Modes

Money fails when it becomes the goal instead of a tool.  
Time fails when it is unstructured and wasted.

### 9. Make Conditional Judgment

Do not say:

`Money is better than time.`

Say:

`Money protects time when survival is unstable. Time becomes the higher-order value when survival is stable.`

### 10. Compress

Final line:

**Money controls options. Time contains life.**

## Required Output

Every comparison must produce:

- A
- B
- lens
- context
- A root
- B root
- A causal chain
- B causal chain
- hidden similarity
- hidden difference
- failure modes
- conditional verdict
- final line

## Publish Gate

Do not publish a comparison unless it has:

1. clear lens
2. clear hidden difference
3. conditional judgment
4. memorable final line
